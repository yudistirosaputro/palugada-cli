How keys are stored

Tokens are written to ~/.palugada/secrets.yaml with 0600 permissions and are never sent back to the browser — you only ever see •••• (N chars). Leaving a key field blank keeps the existing value. Provider & base URL are saved as global defaults; each project still picks its own repo and can override a provider if it needs to.
